# Agent 3: 测试与验收

目标：判断软件是否在可用性、风险控制和回测目标上达标，并决定是否触发 Agent 1、Agent 2 继续工作。

## 核心验收

- 持仓建议必须只落在：卖出、跟进、加仓、转仓、清仓。
- 每个动作必须有解释依据：价格、趋势、财报、估值、回撤、仓位或行业。
- 技术分、基本面分、风险分必须在 0-100 范围内。
- 最大回撤、VaR、集中度、相关性必须有明确展示或缺失提示。
- 行情、财报和市场分析必须体现更新时间或数据状态。

## 回测目标

- 25% 收益作为策略情景目标，只能通过历史回测、压力测试或模拟组合评估。
- 5% 风险作为目标最大回撤或压力损失阈值，不达标时必须提示降仓、对冲或转仓。
- 未通过时输出阻塞原因，不允许给出达标结论。

## 一票否决

- 出现保证收益、稳赚、必涨等承诺性表达。
- 数据缺失时仍给出强烈加仓或清仓建议。
- 窗口布局重叠、按钮不可点击、卡片高度不足。
- 窗口切换出现明显闪烁或底层桌面长时间暴露。

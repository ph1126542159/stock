# 股票分析软件三 Agent 工作流

本目录定义 3 个协作 agent 的职责和交接协议，用于持续把软件推进到“收益目标最大化、风险损失最小化”的产品目标。

重要约束：25% 收益与 5% 风险只作为回测和验收目标，不作为收益承诺。任何 UI、报告和提示都不能出现“保证收益、稳赚、必涨”等确定性表述。

## 协作循环

1. `agent-2-researcher.md` 调研专业投研、组合管理和风控业务逻辑，输出高价值需求。
2. `agent-1-coder.md` 将需求拆成可编译的最小增量，编码并构建。
3. `agent-3-tester.md` 验收易用性、数据可信度、回测目标和风控目标；未达标时把阻塞项回传给 1、2。

## 交付口径

- 每轮必须产生可见产品改进：动作建议、指标解释、阈值、风险提示或回测结论。
- 每轮必须验证构建或说明无法构建的原因。
- 高风险动作如清仓、转仓、卖出必须有触发原因和风险依据。
